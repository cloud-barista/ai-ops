# 외부 연동 책임 경계

## 1. 목적

본 문서는 경희대학교 AI App Deployer가 외부 기관 및 시스템과 연동할 때의 책임 경계를 정의한다. 본 문서는 실제 외부 플랫폼 내부 API 구현 범위를 확장하지 않으며, 1차년도에는 REST/OpenAPI 기반 제공 인터페이스와 Adapter 교체 지점을 명확히 하는 것을 목적으로 한다.

## 2. 경희대학교 제공 범위

```text
- AI App Deployer REST API
- OpenAPI/Swagger 계약
- App 등록/조회 API
- Runtime/Target Profile 등록/조회 API
- Resource Check / Resource Inventory API
- Deployment 생성/상태/로그/중지 API
- Monitoring summary/runtime-health/alarms/metrics API
- 표준 상태 Enum 및 ErrorResponse
- 예제 요청/응답 JSON
- Interface smoke/contract test 기준
```

## 3. 기관별 책임 경계

| 기관/시스템 | 경희대학교 제공 | 상대 기관 확정 필요사항 | 1차년도 처리 |
| --- | --- | --- | --- |
| ETRI | Target Profile, Resource Check, ETRI AI-Infra Adapter skeleton, 표준 error_code 매핑 | AWS GPU VM, 3종 CSP VM, 통합 시험 VM, AI-Infra API, API Gateway 정책 | Mock/Fixture와 Contract Test 우선 |
| 이노그리드 | App 등록 API, 배포 요청 API, 상태/로그 조회 API | App 등록/배포 데이터 매핑, 호출 주체, 실패 처리 방식 | REST API 계약과 예제 제공 |
| 베스핀글로벌 Web Console | App/Deployment/Monitoring 조회 API | Console 화면 action과 API 호출 파라미터 | OpenAPI 기반 연동 |
| Bespin MCP-like API | 배포 요청, 상태 조회, 로그 조회, 중지 API | Tool 입력 schema와 REST API mapping | 호출 시나리오와 에러 매핑 제공 |
| API Gateway | `/api/v1` API, healthz/readiness | 인증, 라우팅, timeout, retry, path rewrite | 설정 항목 문서화 |

## 4. 본 문서 기준 제외 범위

```text
- 실제 ETRI AI-Infra 내부 API 구현
- 실제 Innogrid 내부 플랫폼 API 구현
- 실제 Bespin Web Console/MCP 내부 구현
- Docker/Kubernetes/Container 기반 배포
- Kubernetes Node/Pod capacity 평가
- LLM 운영관리
- Agent Registry / Agent Interface
- 자연어 기반 배포 요청 처리
- 추론 최적화 전략
```

## 5. 외부 API 계약 확정 후 처리 방식

외부 API 계약이 확정되면 다음 순서로 반영한다.

```text
1. external adapter interface 검토
2. 외부 API request/response mapping 작성
3. 표준 Deployment status와 error_code 매핑
4. Mock/Fixture 업데이트
5. Contract Test 작성
6. Go client 구현
7. OpenAPI/가이드/시험 증적 갱신
```

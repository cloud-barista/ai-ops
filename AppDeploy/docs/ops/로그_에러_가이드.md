# 로그·에러 가이드

AI App Deployer는 산출물 문서를 늘리는 대신, 실행 로그와 표준 에러 메시지로 동작을 설명한다.

## 목표

- API 요청부터 Runtime Adapter 실행까지 추적 가능해야 한다.
- 실패 원인이 표준 error_code와 사람이 읽을 수 있는 message로 남아야 한다.
- 제출 증적은 긴 설명 문서보다 요청/응답, DeploymentEvent, Runtime Adapter log, ErrorResponse를 우선한다.

## 필수 로그 필드

| 필드 | 설명 |
| --- | --- |
| `timestamp` | 이벤트 발생 시각 |
| `level` | `INFO`, `ERROR` |
| `request_id` | API 요청 추적 ID |
| `deployment_id` | 배포 단위 추적 ID |
| `component` | `orchestrator`, `validator`, `resource-matcher`, `cpu-vm-adapter`, `gpu-vm-adapter`, `runtime-adapter` 등 |
| `stage` | 표준 Deployment 상태값 |
| `message` | 사람이 읽을 수 있는 작업/실패 설명 |
| `error_code` | 실패 시 표준 에러 코드 |
| `retryable` | 재시도 가능 여부 |

## Deployment 이벤트 기준

| 단계 | 반드시 남길 메시지 |
| --- | --- |
| `REQUESTED` | 배포 요청 생성 |
| `VALIDATING` | App, Runtime Profile, Target Profile 검증 시작 |
| `VALIDATED` | 검증 통과 |
| `SCHEDULING` | 자원 매칭 시작 |
| `DEPLOYING` | Runtime Adapter 배포 요청 |
| `RUNNING` | 앱 healthcheck 또는 실행 확인 |
| `STOPPING` | 중지 요청 수신 |
| `STOPPED` | 원격 프로세스 종료 또는 중지 완료 |
| `*_FAILED` | 실패 원인, error_code, retryable 포함 |

## ErrorResponse 기준

모든 API 에러는 아래 구조를 따른다.

```json
{
  "request_id": "req-...",
  "error": {
    "code": "RUNTIME_FAILED",
    "message": "gpu vm target requires vm.credential_ref",
    "details": {},
    "retryable": false
  }
}
```

## 메시지 작성 원칙

- 어떤 component가 무엇을 하다가 실패했는지 드러나야 한다.
- 사용자가 다음 조치를 알 수 있도록 누락 필드나 실패 조건을 직접 적는다.
- 외부 API 실패는 provider, timeout, auth, gateway failure를 표준 코드로 정규화한다.
- 민감정보는 기록하지 않는다.

## 마스킹 대상

- SSH private key path의 실제 파일명
- API token, password, secret
- 내부 전용 host, endpoint, credential 원문
- 원문 외부 API 응답에 포함된 인증 정보

## 좋은 메시지 예시

| 상황 | 메시지 |
| --- | --- |
| App Spec 오류 | `artifact.type must be one of package, git, binary, script` |
| Target Profile 오류 | `gpu vm target requires vm.credential_ref` |
| GPU readiness 실패 | `nvidia-smi not found` |
| 배포 실패 | `ssh command failed for target target-gpu-001` |
| 중지 완료 | `gpu vm process stop requested: stopped` |
| 이미 종료됨 | `gpu vm process stop requested: no-process` |

## 제출 증적 우선순위

1. API 요청/응답 JSON
2. `/api/v1/deployments/{deployment_id}/logs`
3. `go test ./...`, `go vet ./...`
4. OpenAPI lint/build 결과
5. VM readiness, inference proxy, stop process 수동 검증 로그
6. 요약 문서

요약 문서는 증적을 대체하지 않는다. 요약 문서는 어떤 로그와 에러를 보면 되는지 안내하는 역할만 한다.

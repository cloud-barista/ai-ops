# 설치 활용 가이드 초안

## 개발 환경
- Ubuntu VM
- Go 1.22 이상 권장
- GitHub 저장소 기반 관리
- Echo 기반 Go 서버

후속 개발과 시험 우선순위는 `docs/planning/경희대_1차년도_개발_백로그.md`를 기준으로 관리한다.
제출 직전 점검은 `docs/release/1차년도_제출_패키지_체크리스트.md`, 증적 구성은 `docs/evidence/증적_패키지_가이드.md`, 외부 API 경계는 `docs/external/외부_연동_경계_정리.md`를 따른다.

## 기본 실행
```bash
git clone <repository-url>
cd ai-app-deployer
go mod tidy
go run ./cmd/server
```

서버 실행 후 API 문서는 다음 경로에서 확인한다.

```text
http://localhost:8080/swagger
http://localhost:8080/openapi.yaml
```

## GPU VM readiness 확인
```bash
nvidia-smi
```

## 환경변수 예시
```bash
export AIAPP_SERVER_PORT=8080
export AIAPP_CONFIG_PATH=./config/dev.yaml
export AIAPP_LOG_LEVEL=DEBUG
export AIAPP_STORE_PATH=./data/aiapp-store.json
```

`AIAPP_STORE_PATH`를 설정하면 App, Runtime/Target Profile, Deployment, Event, Resource Inventory가 JSON 파일로 저장된다.
설정하지 않으면 기본값은 메모리 저장소이다.

## CPU VM SSH Runner 예시
기본 CPU VM Runner는 dry-run이다. 실제 SSH 실행을 사용할 때만 다음처럼 설정한다.

```bash
export AIAPP_CPUVM_RUNNER=ssh
export AIAPP_CREDENTIAL_CRED_LOCAL_CPU_VM_001_SSH_USER=ubuntu
export AIAPP_CREDENTIAL_CRED_LOCAL_CPU_VM_001_SSH_KEY_PATH=/secure/path/cpu-vm-001.pem
export AIAPP_CREDENTIAL_CRED_LOCAL_CPU_VM_001_SSH_TIMEOUT=30s
```

`cred://local/cpu-vm-001`은 환경변수에서 `CRED_LOCAL_CPU_VM_001`로 정규화된다.

`artifact.type=script`이고 `artifact.uri=file://...`이면 로컬 스크립트가 SSH로 대상 VM의
`storage.artifact_dir/{app_name}/{version}` 경로에 업로드된 뒤 해당 경로에서 `entrypoint`가 실행된다.

현재 smoke 예제는 다음 파일이다.

- CPU: `examples/cpu-smoke-run.sh`
- GPU: `examples/gpu-smoke-run.sh`

AI Application 배포 방식은 별도 샘플 앱으로 실제 VM에서 테스트 완료했다. 나중에 실제 AI Application을 배포할 때는 동일한 구조로 모델 서버 실행 스크립트나 실행 바이너리를 `artifact.uri=file://...`로 지정하고, `entrypoint`에 실행 명령을 넣어 배포한다.

## GPU VM SSH Runner 예시
GPU VM은 `nvidia-smi` readiness를 확인한 뒤 배포한다. 실제 SSH 실행을 사용할 때만 다음처럼 설정한다.

```bash
export AIAPP_GPUVM_RUNNER=ssh
export AIAPP_CREDENTIAL_CRED_GPU_VM_001_SSH_USER=ubuntu
export AIAPP_CREDENTIAL_CRED_GPU_VM_001_SSH_KEY_PATH=/secure/path/gpu-vm-001.pem
export AIAPP_CREDENTIAL_CRED_GPU_VM_001_SSH_TIMEOUT=30s
```

Target Profile은 `runtime.runtime_type=gpu`, `runtime.accelerator=nvidia`, `gpu.count >= 1`을 포함해야 한다.

## AI Application 배포 방식

실제 SSH runner를 사용할 때는 Target Profile의 `credential_ref`와 환경변수 credential을 먼저 맞춘다.

| 단계 | 내용 |
| --- | --- |
| 1 | Runtime Profile과 Target Profile을 등록한다. |
| 2 | App Spec에 `artifact.uri`, `entrypoint.command`, `entrypoint.args`, `network.ports`를 지정한다. |
| 3 | Deployment를 생성해 대상 VM의 `storage.artifact_dir/{app_name}/{version}` 경로에 artifact를 업로드하고 실행한다. |
| 4 | 앱이 HTTP endpoint를 제공하면 inference proxy로 health/invoke를 확인한다. |
| 5 | Stop API로 원격 VM의 배포 프로세스를 종료한다. |

CPU VM 배포, GPU VM 배포, inference proxy, stop process 종료 흐름은 테스트 완료했다.

배포 종료는 다음 API를 사용한다. CPU/GPU VM Adapter는 원격 VM의 배포 작업 디렉터리에서 `.aiapp.pid`를 우선 확인하고, 없으면 작업 디렉터리 기반 프로세스를 찾아 종료한다.

```powershell
Invoke-RestMethod -Method Post `
  -Uri http://localhost:8080/api/v1/deployments/{deployment_id}/stop
```

## 모니터링 확인
Deployment와 Resource Check를 수행한 뒤 다음 API로 운영 상태를 확인한다.

```bash
curl http://localhost:8080/api/v1/monitoring/summary
curl http://localhost:8080/api/v1/monitoring/runtime-health
curl http://localhost:8080/api/v1/monitoring/alarms
curl http://localhost:8080/api/v1/monitoring/metrics
```

## AI-Infra 외부 연동 골격
실제 외부 API 연결은 아직 합의 전이므로 기본 서버는 ETRI Mock/Fixture 클라이언트를 사용한다.

- Runtime Profile: `runtime_type=aiinfra`, `adapter_type=etri_aiinfra`, `operating_mode=remote_api`
- Target Profile: `runtime.runtime_type=aiinfra`, `runtime.operating_mode=remote_api`
- `/api/v1/resources/check`, `/api/v1/deployments`, `/api/v1/deployments/{deployment_id}/logs`, `/api/v1/deployments/{deployment_id}/stop` 흐름은 기존 API를 그대로 사용한다.
- 실제 ETRI/이노그리드/베스핀글로벌 API 명세가 확정되면 `internal/external/*` 클라이언트를 교체한다.

## 주의
- SSH Key, Token, Cloud Credential은 환경변수 또는 secret store에서 주입한다.
- credential 값은 문서, 예제, 로그에 기록하지 않는다.
- Docker/Kubernetes 기반 설치 절차는 1차년도 가이드에 포함하지 않는다.

## Smoke/Evidence Script

PowerShell 환경에서는 다음 스크립트로 기본 API 흐름과 증적 수집을 실행한다.

```powershell
.\scripts\api-smoke.ps1 -BaseUrl http://localhost:8080
.\scripts\interface-smoke.ps1 -BaseUrl http://localhost:8080
.\scripts\collect-evidence.ps1 -Port 18080
```

- `api-smoke.ps1`는 실행 중인 서버를 대상으로 healthz, readiness, app/profile 등록, resource check, deployment, metric, logs, stop, monitoring을 확인한다.
- `interface-smoke.ps1`는 외부 제공 인터페이스 예제인 `examples/interface/requests`를 사용해 직접 App Spec 등록, `dry_run` Runtime Profile, GPU Target, 로그/metric/monitoring 흐름을 확인한다.
- `collect-evidence.ps1`는 서버를 dry-run으로 임시 실행하고 `go test`, `go vet`, API smoke, readiness, monitoring 결과를 `test/results/{timestamp}`에 저장한다.
- 요청 body 예제는 `examples/requests`에서 확인한다.
- Metric 확인은 `GET /api/v1/monitoring/metrics` 또는 `GET /api/v1/deployments/{deployment_id}/metrics`를 사용한다.
- 최신 제출 후보 증적은 `test/results/20260625-173058`를 기준으로 확인할 수 있다.

# 시험 가이드 초안

## 시험 원칙
- 모든 시험에는 TC ID를 부여한다.
- API 요청/응답, request_id, deployment_id, 로그 파일, VM/GPU readiness 결과를 보존한다.
- 사람 검증자가 실제 VM/GPU/외부 API 통합 시험을 최종 확인한다.

## 로그 수집
```bash
curl http://localhost:8080/api/v1/deployments/{deployment_id}/logs
```

## Inference/Stop 수동 검증

HTTP inference endpoint를 제공하는 앱은 control-plane API로 health와 invoke를 확인한다. Stop 검증은 API 응답의 `status=STOPPED`, deployment log의 `STOPPING`/`STOPPED`, 원격 VM의 app process 제거, app port 연결 실패를 함께 확인한다.

Inference proxy와 stop process 종료 흐름은 테스트 완료했다.

## Evidence Collection

PowerShell 환경에서는 다음 명령으로 테스트와 API smoke 결과를 수집한다.

```powershell
.\scripts\collect-evidence.ps1 -Port 18080
```

외부 제공 인터페이스 예제 검증은 서버 실행 후 다음 명령으로 별도 수행한다.

```powershell
.\scripts\interface-smoke.ps1 -BaseUrl http://localhost:8080
```

결과는 `test/results/{timestamp}`에 저장하며, 주요 파일은 다음과 같다.

- `go-test.txt`
- `go-vet.txt`
- `api-smoke.txt`
- `api-smoke/*.json`
- `readiness.json`
- `monitoring-summary.json`
- `monitoring-metrics.json`
- `manifest.json`

제출 증적 패키지 구성과 최종 판정 기준은 `docs/evidence/증적_패키지_가이드.md`를 따른다.

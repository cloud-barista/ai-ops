# 시험 증적 패키지 가이드

작성일: 2026-06-25

## 목적

본 문서는 1차년도 CPU/GPU VM 기반 AI 응용 등록·배포 프로토타입이 실제로 동작했다는 근거를 제출 가능한 형태로 정리하기 위한 기준이다.

증적은 “기능이 있다”는 설명보다 강한 자료이므로, API 요청·응답, 테스트 결과, readiness 결과, 로그, 실패 매핑을 함께 보존한다.

산출물 문서는 최소화하고, 실제 판단은 로그와 에러 메시지를 우선한다. 로그·에러 작성 기준은 `docs/ops/로그_에러_가이드.md`를 따른다.

## 수집 명령

PowerShell 환경에서 다음 명령을 실행한다.

```powershell
.\scripts\collect-evidence.ps1 -Port 18083
```

결과는 `test/results/{timestamp}`에 저장된다. 최신 기준 증적은 `test/results/20260625-173058`이며, `manifest.json` 기준으로 `go_test_exit_code`, `go_vet_exit_code`, `api_smoke_exit_code`가 모두 0이다.

2026-07-01에는 별도 수동 검증으로 NHN Cloud 원격 VM에서 CPU/GPU VM 기반 AI Application 배포 방식, inference proxy 호출, stop API 기반 원격 프로세스 종료를 확인했다. 샘플 앱은 최종 서비스 앱이 아니라 배포 방식 검증 용도로만 사용했다. 수동 검증 요약은 `log.md`를 기준으로 관리하고, 최종 제출 시에는 SSH key나 내부 credential 값을 제거한 별도 증적 파일로 정리한다.

## 현재 증적 구성

| 파일/폴더 | 설명 |
| --- | --- |
| `manifest.json` | 증적 생성 시각, base URL, output dir, test exit code, store path |
| `go-test.txt` | `go test ./...` 실행 결과 |
| `go-vet.txt` | `go vet ./...` 실행 결과 |
| `api-smoke.txt` | API smoke script 실행 로그 |
| `api-smoke/*.json` | API별 요청/응답 결과 |
| `readiness.json` | 서버 readiness 결과 |
| `monitoring-summary.json` | 모니터링 요약 API 결과 |
| `monitoring-metrics.json` | Metric placeholder 조회 결과 |
| `aiapp-store.json` | smoke 실행 중 생성된 파일 저장소 snapshot |
| `server.stdout.txt`, `server.stderr.txt` | 증적 수집 중 API 서버 출력 |

## 민감정보 처리

- SSH private key, API token, cloud credential은 증적에 포함하지 않는다.
- Target Profile에는 실제 secret 대신 `credential_ref`만 남긴다.
- 외부 API 원문 응답을 저장할 때는 token, key, endpoint 내부 경로를 제거한다.
- 발표자료에는 공개 가능한 host/IP만 사용하고, 필요하면 masked value로 대체한다.

## 미제공 환경 처리

실제 VM, GPU, 외부 API가 제공되지 않은 경우 실패로 보지 않고, 아래 정보를 증적에 남긴다.

| 상황 | 기록 방식 |
| --- | --- |
| GPU VM 미제공 | GPU dry-run 결과와 “실 GPU VM 제공 대기” 사유 기록 |
| ETRI API 미확정 | mock/fixture 결과와 API 계약 대기 사유 기록 |
| Innogrid/Bespin 연동 미확정 | OpenAPI 호출 경계와 책임 분리 문서 첨부 |
| API Gateway 인증 미확정 | `GATEWAY_AUTH_FAILED` 매핑 기준과 설정 대기 사유 기록 |

## 최종 판정 기준

제출 후보는 아래 조건을 만족해야 한다.

- `go test ./...` PASS
- `go vet ./...` PASS
- API smoke PASS
- Swagger/OpenAPI 조회 가능
- TC별 대표 증적 파일 존재
- 실 API 미구현 사유와 교체 지점 문서화
- 1차년도 제외 범위가 코드와 문서에 섞이지 않음

# 경희대학교 1차년도 개발 백로그

작성일: 2026-06-25

## 목적

본 문서는 경희대학교 담당 범위에서 현재 구현된 `AI App Deployer` 프로토타입 다음에 수행할 개발·문서·시험 작업을 정리한다.

현재 코드는 CPU/GPU VM 기반 AI 응용 등록·배포 흐름을 대략 구현한 상태이므로, 이후 작업은 단순 기능 추가보다 1차년도 산출물 수용 기준에 맞춰 정렬한다.

| 영역 | 상태 | 비고 |
| --- | --- | --- |
| Go/Echo API 서버 | 구현됨 | `/api/v1` prefix, request_id middleware, 공통 ErrorResponse |

| Runtime/Target Profile | 구현됨 | mock, cpu, gpu, aiinfra profile 등록 |
| Deployment Orchestrator | 구현됨 | REQUESTED → RUNNING, STOPPING → STOPPED 전이와 이벤트 기록 |

| CPU VM Adapter | 구현됨 | dry-run, SSH runner, local `file://` script upload |
| GPU VM Adapter | 구현됨 | `nvidia-smi` readiness, dry-run, SSH runner |
| AI App 배포 방식 검증 | 구현/검증됨 | CPU/GPU VM 배포 방식 테스트 완료 |

## 바로 다음 구현 순서

1. 프레임워크 공유 프롬프트를 기준으로 후속 작업 지시를 통일한다.
2. 로그·에러 가이드 기준으로 증적을 다시 수집하고 최종 제출 폴더를 지정한다.
3. inference/stop 수동 검증 결과를 credential 제거 후 제출 증적에 포함한다.
4. 공식 DOCX 산출물이 필요하면 최소 변경만 반영한다.
5. 최종 release candidate tag와 제출 패키지 압축본을 생성한다.

## 1차년도 제외 유지

- Docker, Docker Compose, Kubernetes, OCI Image, Container Registry 기반 배포 구현
- GPU/TPU/NPU 컨테이너 기반 배포 자동화
- 실제 AI 반도체 VM/NPU Runtime 직접 제어
- LLM 운영관리/에이전트 등록관리/추론 최적화 전략을 `AI App Deployer` 배포 코드에 무리하게 결합하는 것

## 판단

현재 가장 중요한 일은 배포 기능을 더 복잡하게 만드는 것이 아니라, 경희대 담당 범위의 1차년도 산출물을 제출 가능한 패키지로 묶는 것이다. 따라서 다음 작업은 `최종 증적 재수집`, `구조 설계서 최신화`, `프로토타입 개발설계서 최신화`, `API/설치/시험 가이드 동기화`, `release candidate 패키징` 순서로 진행한다.

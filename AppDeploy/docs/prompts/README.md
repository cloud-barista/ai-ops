# Framework Prompt Guide

이 디렉터리는 AI App Deployer를 개발·검증·공유할 때 사용할 프레임워크 프롬프트를 관리한다.

문서 산출물을 많이 늘리기보다, 동일한 프롬프트와 로그·에러 기준으로 여러 작업자가 같은 방향으로 개발하게 만드는 것이 목적이다.

## 핵심 문서

| 문서 | 용도 |
| --- | --- |
| `프레임워크_공유_프롬프트.md` | 다른 에이전트나 작업자에게 그대로 전달할 기준 프롬프트 |
| `../../agent_md/00_scope_common_contract.md` | 범위, 제외 항목, 상태값, 에러 코드 |
| `../../docs/ops/로그_에러_가이드.md` | 로그와 에러 메시지 작성 기준 |

## 운영 원칙

- 먼저 프롬프트를 공유하고, 산출물 문서는 최소화한다.
- 구현 설명보다 API 계약, 로그, 에러, 시험 증적을 우선한다.
- 기능별 상세 문서는 새로 만들기보다 기존 OpenAPI, agent_md, 로그/에러 가이드에 연결한다.
- 작업 완료 후에는 `log.md`에 짧게 기록한다.

## 기록 항목

프롬프트 기반 작업 기록이 필요할 때만 아래 항목을 남긴다.

- 입력 요구사항
- 사용한 기준 프롬프트
- 참조 문서
- 변경 요약
- 검증 결과
- 남은 판단 또는 외부 합의 대기 항목

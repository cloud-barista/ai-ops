# 1차년도 제출 패키지 체크리스트

작성일: 2026-06-25
최근 정리일: 2026-07-01

## 목적

본 문서는 경희대학교 담당 범위인 `AI 반도체 기반 AI 응용 배포 및 운용 구조 설계`와 `CPU/GPU VM 기반 AI 응용 등록·배포 프로토타입 개발`을 제출 가능한 패키지로 정리하기 위한 최종 점검표이다.

기능을 추가하기보다 현재 구현 기준선, 문서, OpenAPI, 시험 증적, 외부 연동 경계를 서로 맞추는 것을 우선한다.

## 정리 방향

| 원칙 | 적용 방식 |
| --- | --- |
| 프레임워크 프롬프트 중심 | `docs/prompts/프레임워크_공유_프롬프트.md`를 공유 기준으로 사용 |
| 산출물 문서 최소화 | 구조 설계서, 프로토타입 개발설계서, OpenAPI, 설치/시험/증적/릴리스 문서만 전면 유지 |
| 로그 및 에러 메시지 최대화 | `request_id`, `deployment_id`, `stage`, `component`, `error_code`, `retryable`, `message`를 증적 중심으로 관리 |

요약 문서는 증적을 대체하지 않는다. 제출 판단은 API 계약, DeploymentEvent, Runtime Adapter 로그, ErrorResponse, 시험 결과를 기준으로 한다.

## 최종 제출 전 남은 일

- `.\scripts\collect-evidence.ps1 -Port 18083`를 최종 기준선에서 다시 실행한다.
- `test/results/{timestamp}` 중 최종 제출용 폴더를 하나로 지정한다.
- Swagger 화면, API smoke 결과, VM/GPU readiness 로그, inference/stop 수동 검증 결과를 발표자료에 첨부한다.
- SSH key, token, cloud credential, host 내부정보가 증적에 포함되지 않았는지 확인한다.
- 공식 DOCX 산출물이 필요한 경우 최신 Markdown 변경 내용을 DOCX에 반영한다.
- 최종 release candidate tag를 생성하고 제출 패키지명에 반영한다.

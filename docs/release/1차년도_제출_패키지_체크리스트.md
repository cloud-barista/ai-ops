# 1차년도 제출 패키지 체크리스트

## 1. 목적

이 체크리스트는 제출 전 Go 기반 service-control prototype, 공식 산출물, 검증 증적이 서로 맞는지 확인하기 위한 최종 점검표입니다.

## 2. 코드 범위

| 확인 | 항목 | 기준 |
| --- | --- | --- |
| [ ] | Go 중심 구현 | 핵심 기능이 `go/service-control-api`, `go/aiops-guard`에 위치 |
| [ ] | Agent registry | `config/agent_registry.json`과 Go validation logic 일치 |
| [ ] | LLM selection config | `config/ops_llm_benchmark.json`이 role label과 actual model 경계를 분리 |
| [ ] | CPU/GPU placement config | `config/inference_optimization.json`에 workload/resource profile 존재 |
| [ ] | 실행 결과 제외 | `runs/`, credential, cache, build output이 Git 추적 대상이 아님 |

## 6. LLM 평가 상태

| 확인 | 상태 | 기준 |
| --- | --- | --- |
| [ ] | 기본 정책 baseline | `benchmark_status = not_executed`를 실제 benchmark로 표현하지 않음 |
| [ ] | Dry-run | `benchmark_status = dry_run`은 wiring 검증으로만 표현 |
| [ ] | 실제 실행 | 실제 모델 평가를 주장하려면 `benchmark_status = executed` 증적 필요 |
| [ ] | Endpoint 실패 | endpoint 미실행 실패는 숨기지 않고 evidence로 보존 |

## 7. VM 검증 상태

| 확인 | 항목 | 기준 |

## 8. 최종 제출 전 금지 표현

| 금지 표현 | 대체 표현 |
| --- | --- |
| 실제 LLM 최종 성능을 입증했다 | 실제 endpoint 실행 결과가 있는 경우에만 executed benchmark로 기술 |
| 운영 환경 배포 완료 | 1차년도 Go 기반 기능 프로토타입 검증 |
| GPU VM 자동 생성 기능 구현 | CPU/GPU VM 기반 배치 판단 및 배포·제어 계획 생성 |
| 완성형 AIOps 플랫폼 | 서비스 제어 및 관리 자동화 프레임워크 prototype |

# 로그·에러 가이드

## 1. 목적

이 문서는 Go 기반 service-control prototype에서 생성되는 상태값, 검증 로그, 오류 메시지를 어떻게 해석하고 보고서에 표현할지 정의합니다.

## 2. 주요 상태값

| 상태값 | 위치 | 의미 |
| --- | --- | --- |
| `valid = true` | validation summary | 해당 단계의 입력, policy, guard 검증이 통과 |
| `valid = false` | validation summary | 해당 단계가 실패했으며 error/detail 확인 필요 |
| `benchmark_status = not_executed` | LLM selection/benchmark | 실제 LLM 호출이 수행되지 않음 |
| `benchmark_status = dry_run` | LLM benchmark | scenario/candidate/evaluator 연결만 검증 |
| `benchmark_status = executed` | LLM benchmark | 실제 OpenAI-compatible endpoint 응답을 evaluator가 처리 |
| `guard_validation.valid = true` | service operations | 요청 action이 bounded action 정책을 통과 |

## 3. 자주 보는 실패 원인

| 오류/현상 | 원인 | 조치 |
| --- | --- | --- |
| `connect: connection refused` | LLM endpoint가 실행 중이 아님 | endpoint 실행 후 다시 benchmark |
| `no enabled LLM candidate executed successfully` | enabled candidate가 모두 호출 실패 | candidates config와 endpoint 상태 확인 |
| `benchmark_status = not_executed` | 실제 LLM 호출이 없거나 후보 비활성화 | 실제 평가로 표현하지 않음 |
| `nvidia-smi: command not found` | GPU driver/tool 미설치 또는 GPU VM이 아님 | VM image/driver/GPU visibility 확인 |
| `metadata request failed` | VM 내부가 아니거나 metadata 접근 불가 | `--target vm` 실행 위치 확인 |
| `agent action is not allowed` | action이 registry의 bounded action 밖 | `config/agent_registry.json` 확인 |
| `workload not found` | 요청 workload ID가 config에 없음 | `config/inference_optimization.json` 확인 |

## 4. 로그 보존 방식

권장 폴더명:

```text
runs/validation-YYYYMMDD-HHMMSS/
```

권장 포함 파일:

| 파일 | 내용 |
| --- | --- |
| `00_system_validation_summary.json` | 전체 단계 성공/실패 요약 |
| `01_environment.json` | 실행 환경 |
| `02_go_test_aiops_guard.txt` | guard test output |
| `03_go_test_service_control_api.txt` | service-control test output |
| `team-validation/` | 기능별 JSON 증적 |
| `ops-llm-benchmark/` | LLM benchmark output/evaluation |

## 5. 보고서 표현 기준

| 증적 상태 | 보고서 표현 |
| --- | --- |
| `not_executed` | LLM 후보 선정 구조와 정책 baseline을 구현함 |
| `dry_run` | Ops LLM 평가 파이프라인과 evaluator 연결을 검증함 |
| `executed` | 실제 endpoint 기반 Ops LLM 응답 평가를 수행함 |
| endpoint 연결 실패 | 실제 실행을 시도했으나 endpoint 미준비로 실패했음을 명시함 |
| VM GPU 미탐지 | VM 실행 환경의 GPU driver 또는 runtime 준비가 미완료임을 명시함 |

## 6. 사람 검토 기준

- 실패 로그를 성공처럼 요약하지 않습니다.
- `dry_run` 결과를 실제 LLM 성능 평가로 표현하지 않습니다.
- VM 검증 결과는 실행 위치와 GPU visibility 증거를 함께 확인합니다.
- 비용, quota, credential 관련 오류는 코드 오류와 분리해 기록합니다.
